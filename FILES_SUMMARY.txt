BACKEND SERVER (Node.js + Express)
═══════════════════════════════════════════════════════════════════

backend/kalshi-server-oauth.js
  Purpose: Main OAuth authentication server
  What it does:
    - Receives login requests from frontend
    - Sends credentials to Kalshi API (securely from .env)
    - Returns safe session tokens to frontend
    - Provides market data endpoints
    - Executes trades via API
    - Controls trading bot process
  Endpoints:
    POST /api/auth/login - Authenticate with Kalshi
    GET /api/auth/status - Check if authenticated
    POST /api/auth/logout - Logout and cleanup
    GET /api/markets - Get open markets
    POST /api/orders - Place trades
    GET /api/orders - Get order history
    POST /api/bot/start - Start trading bot
    POST /api/bot/stop - Stop trading bot
    GET /api/bot/status - Bot running status

backend/trading-bot.js
  Purpose: Automated trading bot process (subprocess)
  What it does:
    - Spawned by main server when "Start Bot" clicked
    - Polls Kalshi API every 5 seconds for open markets
    - Compares market data against your criteria
    - Automatically executes trades when criteria match
    - Tracks executed trades to avoid duplicates
    - Graceful shutdown on stop signal

backend/package.json
  Purpose: Node.js project configuration
  Contains:
    - Project metadata
    - npm scripts (start, dev, bot)
    - Dependencies:
      * express - Web server framework
      * axios - HTTP client for API calls
      * cors - Enable cross-origin requests
      * dotenv - Load environment variables

HOW FILES WORK TOGETHER
═══════════════════════════════════════════════════════════════════

User Flow:
  1. User opens browser → frontend/index.html loads
  2. React renders → frontend/src/App.jsx
  3. Styling applied → frontend/src/App.css
  4. User logs in → API call to backend/kalshi-server-oauth.js
  5. Backend reads .env (credentials)
  6. Backend authenticates with Kalshi API
  7. Returns session token to frontend
  8. Frontend stores token, shows dashboard
  9. User clicks "Start Bot"
  10. Backend spawns → backend/trading-bot.js process
  11. Trading bot polls markets every 5 seconds
  12. When criteria match → Trades executed via Kalshi API
  13. Frontend displays activity in real-time

Data Flow:
  User Input (Browser)
       ↓
  frontend/src/App.jsx (Process)
       ↓
  backend/kalshi-server-oauth.js (Authenticate & Proxy)
       ↓
  Kalshi API (Execute)
       ↓
  backend/trading-bot.js (Monitor & Trade)
       ↓
  Activity Log (Display Results)

SECURITY SUMMARY
═══════════════════════════════════════════════════════════════════

Who sees what:
  ✅ You: Full code + credentials in .env
  ✅ Backend: Credentials from .env, uses them securely
  ❌ Frontend: Never sees credentials, only session tokens
  ❌ Browser: Never stores passwords, only tokens
  ❌ Network: Always uses HTTPS to Kalshi (secure)

Token Lifecycle:
  1. You provide email/password to frontend
  2. Frontend sends to backend (same machine, safe)
  3. Backend authenticates with Kalshi
  4. Kalshi returns token
  5. Backend returns token to frontend
  6. Frontend uses token for API calls
  7. Token expires after 24 hours
  8. Must re-login to get new token

Files to never commit:
  - .env (contains credentials)
  - node_modules/ (too large)
  - dist/ (can rebuild)
  - .DS_Store (macOS junk)
